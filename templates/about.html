Graph Theory

Graph Theory is a branch of mathematics that studies the properties of graphs — mathematical structures used to model pairwise relations between objects. A graph G = (V, E) consists of a set of vertices (nodes) V and a set of edges E connecting pairs of vertices.

Key Definitions

Degree: The number of edges incident to a vertex. In a directed graph, vertices have in-degree and out-degree.
Path: A sequence of vertices with edges between consecutive vertices, with no repeated vertices.
Cycle: A closed path — starts and ends at the same vertex.
Connected Graph: A graph where there is a path between every pair of vertices.

Graphs can be:

  • Directed (Digraphs): Edges have direction from one vertex to another.
  • Undirected: Edges connect two vertices without direction.
  • Weighted: Each edge carries a numeric value (cost, distance, etc.).
  • Simple: No self-loops or multiple edges between the same vertices.

Euler & Hamilton

Eulerian Path/Circuit — Named after Leonhard Euler's 1736 solution to the Seven Bridges of Königsberg problem, this is the first theorem in graph theory.

Euler's Rules

Eulerian Circuit: Exists iff the graph is connected and every vertex has even degree.
Eulerian Path: Exists iff the graph is connected and exactly two vertices have odd degree (start and end points).
Not Eulerian: More than two vertices with odd degree.

Hamiltonian Path/Cycle — A path (or cycle) that visits every vertex exactly once. Unlike Euler, no simple polynomial-time algorithm is known (it is NP-complete).

Shortest Paths

Dijkstra's Algorithm (1959) finds the shortest path from a source vertex to all other vertices in a weighted graph with non-negative weights. It uses a greedy approach with a priority queue.

Dijkstra Algorithm — Steps

1. Set distance to source = 0, all others = ∞
2. Visit the unvisited vertex with minimum distance
3. Update distances of adjacent vertices via relaxation
4. Repeat until destination is reached

Graph Coloring

Graph coloring assigns colors to vertices such that no two adjacent vertices share the same color. The minimum number of colors needed is the chromatic number χ(G).

Applications: register allocation in compilers, scheduling, map coloring (Four Color Theorem states any planar map needs at most 4 colors).

Trees

A tree is a connected, acyclic undirected graph. Key properties: a tree with n vertices has exactly n-1 edges; there is exactly one path between any two vertices.

  • Rooted Tree: One vertex is designated the root; defines parent-child relationships.
  • Leaf: A vertex with no children (degree 1 in unrooted tree).
  • Height: Length of the longest path from root to a leaf.
  • Huffman Tree: An optimal prefix-free binary tree for data compression.

Algorithms Implemented

Dijkstra's

Single-source shortest path for weighted graphs.

O((V+E) log V)

Eulerian Path

Hierholzer's algorithm for Euler circuits/paths.

O(E)

Hamiltonian

Backtracking search for Hamiltonian paths/cycles.

O(N!)

Greedy Coloring

Largest-first greedy graph coloring.

O(V² + E)

Boyer-Myrvold

Linear-time planarity testing algorithm.

O(V)

Tarjan's

Articulation points and biconnected components.

O(V + E)

Huffman Coding

Optimal prefix codes via min-heap.

O(N log N)

Isomorphism

VF2 algorithm for graph isomorphism testing.

O(N!)

About This Project

This Graph and Tree Analysis Toolkit is built as a Discrete Mathematics project for B.Tech Computer Science (AI & ML). It demonstrates the practical application of graph theory algorithms using Python (NetworkX), Flask, and interactive browser-based visualization.

The backend uses NetworkX — a powerful Python library for graph analysis — combined with Matplotlib for visualization. The frontend uses modern HTML5/CSS3 for a professional, responsive interface.
